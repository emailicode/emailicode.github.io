1. What is MySQL

5. Primary Key

A primary key uniquely identifies each record in a table. Properties:
  • Unique
  • Cannot be NULL
  • Only one per table
Example: 
CREATE TABLE users(
id INT PRIMARY KEY,
name VARCHAR(100)
);

6. Auto Increment

Automatically increases numeric value for each row. 
CREATE TABLE users(
id INT AUTO_INCREMENT PRIMARY KEY,
name VARCHAR(100)
);
Result: | id | name | |----|------| 1 | John 2 | Alice

7. Timestamp

A TIMESTAMP stores date and time values. Commonly used to track: - record creation - updates - logs Example table: 
CREATE TABLE posts(
id INT AUTO_INCREMENT PRIMARY KEY,
title VARCHAR(200),
created_at TIMESTAMP
);
Automatically insert current time: 
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
Update automatically when row changes: 
updated_at TIMESTAMP
DEFAULT CURRENT_TIMESTAMP
ON UPDATE CURRENT_TIMESTAMP
Example result: | id | title | created_at | |----|------|------------| 1 | My Post | 2026-03-11 14:32:00 |

SELECT * FROM users
WHERE id > 2;
Operators: | Operator | Meaning | |---|---| = | equal | > | greater | < | less | != | not equal |

11. Joins

Used to combine data from multiple tables. Example: 
SELECT users.name, orders.price
FROM users
JOIN orders
ON users.id = orders.user_id;
Diagram:
Users Table Orders Table ↓ ↓ JOIN ON user_id ↓ Result Table

12. Indexes

Indexes improve query performance. 
CREATE INDEX idx_email
ON users(email);
Benefits: - Faster search - Faster joins

13. Normalization

Normalization organizes data to reduce redundancy. Bad design: employees(id, name, department_name) Better design: employees(id, name, department_id) departments(id, department_name)

14. Transactions

Transactions ensure reliable operations. Commands: 
START TRANSACTION;
COMMIT;
ROLLBACK;
Example: 
START TRANSACTION;

UPDATE accounts
SET balance = balance - 100
WHERE id = 1;

UPDATE accounts
SET balance = balance + 100
WHERE id = 2;

COMMIT;

15. Query Optimization

Use EXPLAIN to analyze queries. 
EXPLAIN SELECT * FROM users WHERE email='a@email.com';
Best practices: - Use indexes - Avoid SELECT * - Use proper joins - Limit results
